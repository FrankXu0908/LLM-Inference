# Tracing Workflow

## Goal

Create a reproducible path from runtime execution to actionable profiling reports.

`run_pipeline.py` is post-processing only; it requires traces first.

## 1) Trace Types

- `torch_trace_json`
  - Generated by `torch.profiler` (contains `traceEvents`)
  - Used for operator-level attribution in framework/runtime view
- `nsys_raw_rep` / `nsys_raw_qdstrm`
  - Raw Nsight Systems capture artifacts
- `nsys_sqlite_export`
  - NSYS export format used by parser scripts
  - Preferred input for analysis (`parse_nsys.py`)

## 2) Recommended Naming

Use this convention for each capture:

`<model>__<mode>__<phase>__c<concurrency>__in<input_tokens>__out<output_tokens>__run<id>`

Examples:
- `qwen35b__tp2__prefill__c1__in8192__out1__run1.nsys-rep`
- `qwen35b__dp2ep__decode__c16__in256__out128__run3.trace.json`

## 2.1) Standard Directory Layout

All traces should be organized under `results/traces`:

```text
results/traces/
  nsys/
    <phase>/
      <mode>/
        <run_id>/
          *.nsys-rep / *.sqlite / *.qdstrm
  torch/
    <phase>/
      <mode>/
        <run_id>/
          *.json / *.trace.json
```

Where:
- `phase`: `prefill` / `decode` / `mixed`
- `mode`: `tp2` / `dp2` / `dp2_ep` / `unknown`
- `run_id`: stable run identifier, e.g. `run_tp_prefill`

## 3) Capture Workflow

### A. NSYS capture

1. Run service/workload with NVTX enabled.
2. Capture with nsys:

```bash
nsys profile \
  --trace=cuda,nvtx,osrt \
  --force-overwrite=true \
  --output=<prefix> \
  python <your_workload_script>.py
```

3. Export to sqlite:

```bash
nsys export --type sqlite --output <prefix> <prefix>.nsys-rep
```

### B. Torch trace capture

Collect `.json` trace from `torch.profiler` and save under a dedicated trace folder.

## 4) Post-processing

```bash
python benchmark/analysis/profiling/run_pipeline.py \
  --torch-trace-file <path/to/torch_trace.json> \
  --nsys-trace-file <path/to/nsys.sqlite>
```

If one branch is temporarily missing:

```bash
python benchmark/analysis/profiling/run_pipeline.py --skip-missing
```

## 5) Inventory and Hygiene

Build inventory:

```bash
python benchmark/analysis/profiling/classify_traces.py
```

Generated:
- `results/analysis/profiling/trace_inventory.json`
- `results/analysis/profiling/trace_inventory.md`

Use inventory to detect:
- invalid sqlite files
- orphaned raw traces without export
- traces with unclear scenario naming
